# Fix 07 — Removed stale dead files from data/tracklists/

**Severity:** ⚪ LOW
**Files removed:** 10 JSON files, 1 README, 1 Python script
**Status:** ✅ FIXED

## The problem

`data/tracklists/` contained:
- 10 hand-rolled JSON files (May 20-22 dates): `chaos-in-the-cbd.json`, `david-august.json`, `floorplan.json`, `folamour.json`, `helena-hauff.json`, `hunee.json`, `kerri-chandler.json`, `kettama.json`, `mall-grab.json`, `peggy-gou.json`
- A `README.md` that was a copy-paste of `BUILD.md`
- A `extract_tracklists.py` Python script

**None of these are referenced by the active code.** The 3D Brain and DJ Library both load `data/tracklists/tracklists.js`, which is regenerated by `scripts/build-dj-data.js` (and now also by the deploy workflow per Fix 06).

These files were an artifact of an older pipeline (pre-`build-dj-data.js`) that hand-extracted tracklists from individual DJ pages. They've been dead since the migration to the build script.

## The fix

```bash
rm data/tracklists/chaos-in-the-cbd.json
rm data/tracklists/david-august.json
rm data/tracklists/floorplan.json
rm data/tracklists/folamour.json
rm data/tracklists/helena-hauff.json
rm data/tracklists/hunee.json
rm data/tracklists/kerri-chandler.json
rm data/tracklists/kettama.json
rm data/tracklists/mall-grab.json
rm data/tracklists/peggy-gou.json
rm data/tracklists/README.md
rm data/tracklists/extract_tracklists.py
```

Kept `data/tracklists/tracklists.js` (the active compiled file).

## Files changed

- Deleted 12 files in `data/tracklists/`. Folder is now 1 file (the active one).
